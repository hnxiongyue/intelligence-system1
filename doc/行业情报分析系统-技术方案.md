# 行业情报分析系统 - 技术方案

## 一、项目概述

### 1.1 项目背景

当前市场部每周需要人工收集、整理国家政策法规、竞品动态、行业标准等信息，工作量大、时效性差、容易遗漏关键信息。本项目旨在构建一个基于 AI Agent 的自动化情报分析系统，实现：

- **广度**：覆盖国内外政策、竞品、合规等多维度信息源
- **深度**：AI 深度分析，提供影响评估和应对建议
- **时效**：7x24 小时自动监控，第一时间获取关键信息

### 1.2 核心价值

- 替代市场部 70% 的重复性信息收集工作
- 提升情报获取时效性（从周级到小时级）
- 按角色精准推送，提高信息利用率
- 建立企业情报知识库，支持历史追溯和趋势分析

### 1.3 目标用户

- **经营管理者**：关注竞品资本动态、市场趋势
- **产品经理**：关注竞品新功能、用户需求变化
- **项目负责人**：关注政策合规要求、实施建议
- **技术人员**：关注标准规范更新、密码算法动态

### 2.1 整体架构


```
┌─────────────────────────────────────────────────────────────────┐
│                      定时调度层（APScheduler）                    │
│                  每日 9:00 / 每周一 8:00 自动触发                 │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│                   LangGraph 工作流编排层                          │
├─────────────────────────────────────────────────────────────────┤
│  采集 Agent → 清洗 Agent → 分析 Agent → 推送 Agent               │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│                        工具层（可扩展）                           │
├─────────────────────────────────────────────────────────────────┤
│ • MCP Servers：Firecrawl / GitHub / Email                       │
│ • 自定义工具：Playwright / PDF解析 / RSS订阅                     │
│ • LLM：DeepSeek V3（中文）+ GPT-4（英文）                        │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│                        数据存储层                                 │
├─────────────────────────────────────────────────────────────────┤
│ • 向量数据库（Qdrant）：历史情报检索、记忆闭环                   │
```

### 2.2 核心技术选型

| 技术组件 | 选型方案 | 选型理由 |
|---------|---------|---------|
| **Agent 框架** | LangGraph + LangChain | 主流框架，文档完善，适合复杂工作流 |
| **网页采集** | Firecrawl MCP（主）+ Playwright（辅） | MCP 降低开发门槛，Playwright 处理复杂场景 |
| **LLM** | DeepSeek V3 + GPT-4 | 成本低，中英文兼顾 |
| **向量数据库** | Qdrant | 本地部署，Python 友好，支持记忆检索 |
| **关系数据库** | SQLite | 轻量级，无需额外部署 |
| **定时调度** | APScheduler | Python 原生，配置简单 |
| **通知推送** | 钉钉 Webhook + Email | 企业内部常用 |

## 三、系统功能设计

### 3.1 核心功能模块

#### 3.1.1 智能采集模块

**功能描述**：
- 支持多种数据源类型（静态网页、动态网页、PDF、GitHub、RSS）
- 自动识别内容类型，选择合适的采集工具
- 失败重试机制，确保采集成功率
- 并发采集，提升效率

**技术实现**：
```python
# 采集工具路由
def route_crawler(source_type):
    if source_type == "static_web":
        return firecrawl_mcp
    elif source_type == "dynamic_web":
        return playwright_crawler
    elif source_type == "pdf":
        return pdf_parser
    elif source_type == "github":
        return github_mcp
    elif source_type == "rss":
        return rss_parser
```

#### 3.1.2 AI 清洗与结构化模块

**功能描述**：
- 自动提取关键信息（标题、时间、摘要、链接）
- 内容分类（政策/竞品/合规/技术）
- 去重降噪（过滤重复和无效信息）
- 关键词触发（国密 SM4、量子抗性、融资等）

**数据结构**：
```json
{
  "id": "uuid",
  "title": "政策标题",
  "source": "国密局",
  "source_url": "https://...",
  "publish_date": "2026-04-15",
  "category": "政策出台",
  "content": "原始内容",
  "summary": "AI生成的摘要",
  "keywords": ["国密", "SM4", "商用密码"],
  "created_at": "2026-04-15 09:00:00"
}
```

#### 3.1.3 AI 深度分析模块

**功能描述**：
- **影响评估**：分析对公司业务的影响等级（高/中/低）
- **竞品对比**：自动对比我们与竞品的功能差距
- **趋势预测**：基于历史数据预测行业趋势
- **应对建议**：提供可执行的应对方案

**分析维度**：

### 5.2 核心代码示例

#### 5.2.1 LangGraph 主工作流

```python
from langgraph.graph import StateGraph, END
from typing import TypedDict, List

class IntelligenceState(TypedDict):
    """工作流状态"""
    sources: List[dict]          # 数据源列表
    raw_data: List[dict]         # 原始数据
    cleaned_data: List[dict]     # 清洗后数据
    analyzed_data: List[dict]    # 分析结果
    notifications: dict          # 推送结果

def build_main_graph():
    """构建主工作流"""
    workflow = StateGraph(IntelligenceState)
    
    # 添加节点
    workflow.add_node("crawl", crawl_node)
    workflow.add_node("clean", clean_node)
    workflow.add_node("analyze", analyze_node)
    workflow.add_node("notify", notify_node)
    
    # 设置流程
    workflow.set_entry_point("crawl")
    workflow.add_edge("crawl", "clean")
    workflow.add_edge("clean", "analyze")
    workflow.add_edge("analyze", "notify")
    workflow.add_edge("notify", END)
    
    return workflow.compile()

# 使用
app = build_main_graph()
result = app.invoke({"sources": load_sources()})
```

```sql
    keywords TEXT,  -- JSON 数组
    priority TEXT,  -- 高/中/低
    impact_score REAL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 分析结果表
CREATE TABLE analysis (
    id TEXT PRIMARY KEY,
    intelligence_id TEXT,
    impact_analysis TEXT,
    risks TEXT,  -- JSON 数组
    opportunities TEXT,  -- JSON 数组
    suggestions TEXT,  -- JSON 数组
    target_roles TEXT,  -- JSON 数组
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (intelligence_id) REFERENCES intelligence(id)
);

-- 推送记录表
CREATE TABLE notifications (
    id TEXT PRIMARY KEY,
    intelligence_id TEXT,
    role TEXT,
    channel TEXT,  -- dingtalk/email
    status TEXT,  -- success/failed
    sent_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (intelligence_id) REFERENCES intelligence(id)
);

-- 数据源监控表
CREATE TABLE source_monitor (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    source_name TEXT,
    last_crawl_time TIMESTAMP,
    status TEXT,  -- success/failed
    items_count INTEGER,
    error_message TEXT
);
```

## 七、风险与应对

### 7.1 技术风险

| 风险 | 影响 | 概率 | 应对措施 |
|------|------|------|---------|
| 反爬虫限制 | 采集失败 | 中 | 1. 使用代理池<br>2. 降低采集频率<br>3. 使用 Playwright 模拟真人 |

| LLM 分析不准确 | 误报/漏报 | 中 | 1. Prompt 优化<br>2. 人工审核机制<br>3. 持续训练 |
| 数据源变更 | 采集失败 | 高 | 1. 监控告警<br>2. 快速修复机制<br>3. 备用数据源 |

### 7.2 业务风险

| 风险 | 影响 | 概率 | 应对措施 |
|------|------|------|---------|
| 信息过载 | 用户疲劳 | 中 | 1. 优先级过滤<br>2. 角色化推送<br>3. 汇总推送 |
| 误报影响决策 | 业务损失 | 低 | 1. 置信度标注<br>2. 人工复核<br>3. 反馈机制 |
| 依赖性过强 | 系统故障影响业务 | 低 | 1. 降级方案<br>2. 人工备份<br>3. 多渠道验证 |

### 7.3 合规风险

| 风险 | 影响 | 概率 | 应对措施 |
|------|------|------|---------|
| 数据隐私 | 法律风险 | 低 | 1. 只采集公开信息<br>2. 数据脱敏<br>3. 合规审查 |
| 知识产权 | 侵权风险 | 低 | 1. 注明来源<br>2. 合理使用<br>3. 法务审核 |

### 10.1 方案优势

1. **低门槛**：基于 LangGraph 框架，配置驱动，测试工程师可快速上手
