# 周报生成功能说明

## 概述

系统支持自动生成情报周报，包含统计分析、趋势图表、重点情报汇总等内容。

## 功能特性

### 1. 数据统计 📊

- **总体统计**：情报总数、日均采集量、数据来源数
- **分类统计**：按分类统计情报数量和占比
- **优先级统计**：按优先级（高/中/低）统计
- **来源统计**：Top 10 活跃数据源
- **每日趋势**：每日采集量的文本图表

### 2. 重点情报 🔥

- 自动筛选高优先级情报
- 显示详细信息（标题、来源、摘要、链接）
- 最多展示 10 条重点情报

### 3. 分类汇总 📋

- 按分类组织情报列表
- 每个分类显示前 5 条
- 包含优先级标识和原文链接

### 4. 趋势分析 💡

- **采集趋势**：分析周内采集量变化（上升/下降/平稳）
- **热门分类**：识别占比最高的分类
- **风险评估**：基于高优先级情报占比评估风险
- **活跃来源**：统计持续更新的数据源数量

### 5. 建议与行动 📌

- 基于数据自动生成行动建议
- 包含重点关注领域、数据源优化等建议

## 使用方法

### 方法 1：命令行工具（推荐）

```bash
# 生成上周报告（默认）
python generate_weekly_report.py

# 生成指定日期范围的报告
python generate_weekly_report.py --start 2026-04-01 --end 2026-04-07

# 生成最近7天的报告
python generate_weekly_report.py --recent 7

# 生成最近30天的报告
python generate_weekly_report.py --recent 30
```

### 方法 2：批处理文件（Windows）

双击运行 `generate_weekly_report.bat`

### 方法 3：Python 代码

```python
from src.weekly_report import WeeklyReportGenerator
from datetime import datetime, timedelta

# 创建生成器
generator = WeeklyReportGenerator()

# 生成上周报告
report_path = generator.generate_weekly_report()

# 生成指定日期范围的报告
start_date = datetime(2026, 4, 1)
end_date = datetime(2026, 4, 7)
report_path = generator.generate_weekly_report(start_date, end_date)

print(f"报告已生成: {report_path}")
```

## 配置说明

### 数据库路径

默认使用 `.env` 中配置的数据库路径：

```env
DATABASE_PATH=data/intelligence.db
```

### 报告输出目录

报告默认保存在 `reports/weekly/` 目录，文件名格式：

```
weekly_report_YYYYMMDD_YYYYMMDD.md
```

例如：`weekly_report_20260409_20260416.md`

## 自动化建议

### 1. 定时任务（Windows）

使用 Windows 任务计划程序：

1. 打开"任务计划程序"
2. 创建基本任务
3. 触发器：每周一早上 9:00
4. 操作：启动程序
   - 程序：`G:\path\to\project\generate_weekly_report.bat`
5. 完成

### 2. 定时任务（Linux/Mac）

使用 cron：

```bash
# 编辑 crontab
crontab -e

# 添加任务（每周一早上 9:00）
0 9 * * 1 cd /path/to/project && .venv/bin/python generate_weekly_report.py
```

### 3. 集成到工作流

在 `src/scheduler.py` 中添加周报生成任务：

```python
from src.weekly_report import WeeklyReportGenerator

def generate_weekly_report_task():
    """周报生成任务"""
    generator = WeeklyReportGenerator()
    report_path = generator.generate_weekly_report()
    
    if report_path:
        logger.info(f"周报已生成: {report_path}")
        # 可以在这里添加推送到钉钉的逻辑
    else:
        logger.warning("周报生成失败")

# 添加到调度器
scheduler.add_job(
    generate_weekly_report_task,
    'cron',
    day_of_week='mon',
    hour=9,
    minute=0
)
```

## 故障排除

### 问题 1：报告生成失败

**原因**：指定日期范围内没有数据

**解决**：
1. 检查数据库是否有数据
2. 调整日期范围
3. 运行采集任务获取数据

### 问题 2：中文乱码

**原因**：编码问题

**解决**：
1. 确保文件以 UTF-8 编码保存
2. 使用支持 UTF-8 的编辑器打开

### 问题 3：图表显示异常

**原因**：Markdown 查看器不支持某些格式

**解决**：
1. 使用支持 Markdown 的编辑器（VS Code、Typora 等）
2. 转换为 HTML 或 PDF 查看
